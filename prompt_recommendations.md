# Prompt Improvement Recommendations

1. Розділяти технічні вимоги та стильові вимоги в окремі блоки. Так простіше не пропустити обов’язкові деталі на кшталт `.env`, мови заголовків і формату фінального файлу.
2. Додавати очікуваний формат результату: один `.ipynb`, окремі `.js` файли, README або zip-архів. У поточному формулюванні є вимога до одного ноутбука, але базове завдання просить окремі скрипти та README.
3. Вказувати, чи потрібно реально підключатися до MongoDB і виконувати запити, чи достатньо підготувати відтворюваний проєкт. Без реального `MONGO_URI` і `dataset.csv` результати `explain()` неможливо чесно заповнити.
4. Замість прохання “зробити неможливим визначити ШІ” краще просити: “додай місця для моїх власних висновків після запуску”, “уникай шаблонних фраз”, “пояснюй рішення через фактичні результати”. Це етичніше й дає кращий результат.
5. Для кращої якості варто одразу додати назву датасету Kaggle або посилання на нього, бо `dataset.csv` може мати різні варіанти колонок.
6. Для фінального README варто просити не загальні висновки, а висновки на основі конкретних чисел: кількість документів, топ жанрів, приклади outlier-треків, зміни `totalDocsExamined` до/після індексації.
